# 开发准则

## 文档优先
- **必要时参考 docs/ 目录下的设计文档和开发指引**
- 如果当前需求和设计冲突，明确提示用户是否需要修改设计
- 确保所有变更符合现有的技术架构和核心原则

## 设计文档位置
- **设计文档**: `docs/DESIGN.md`
- **开发指引**: `docs/DEVELOPMENT_GUIDELINES.md`
- **测试用例文档**: `TEST_CASES.md`

## 测试实践
- **测试框架**: Playwright (端到端测试)
- **测试目录**: `tests/` 
- **测试命名**: `*.test.js` (所有测试文件必须以此结尾)
- **测试配置**: `playwright.config.js`
- **运行命令**: `npm run test` 或 `npx playwright test`
- **自动服务器**: 配置中包含自动启动Python服务器的功能
- **测试编写**: 参考 `TEST_CASES.md` 了解详细的测试编写指南

## 代码质量
- 遵循现有代码风格和模式
- 保持向后兼容性  
- 充分测试所有变更

## 用户体验
- 优先考虑用户体验和性能
- 确保跨浏览器兼容性
- 维护多语言支持
